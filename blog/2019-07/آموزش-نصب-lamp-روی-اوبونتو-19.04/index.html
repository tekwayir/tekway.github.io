آموزش نصب LAMP روی اوبونتو 19.04
  • Jul 11, 2019
  • 7 دقیقه برای خواندن

لمپ (LAMP) مجموعه‌ای از نرم‌افزارهای آزاد و متن باز که برای میزبانی محتوای وب در محیط لینوکس استفاده میشه. تو این آموزش به شما نشون میدیم که چجوری لمپ رو روی اوبونتو ۱۹.۰۴ نصب کنید.

گام اول: آپدیت کردن پکیج‌های اوبونتو

خب قبل از نصب لمپ بهتره که مخازن و بسته‌های نرم‌افزاری اوبونتو رو آپدیت کنید. برای این کار از دستورات زیر استفاده میکنیم:

sudo apt update
sudo apt upgrade

گام دوم: نصب آپاچی (Apache Web Server)

با دستورات زیر آپاچی رو نصب کنید. بسته‌ی apache2-utils برخی ابزارهای مفید مثل Apache HTTP Server Benchmarking tool رو نصب خواهد کرد.

sudo apt install -y apache2 apache2-utils

بعد از نصب، آپاچی به صورت خودکار شروع به کار خواهد کرد. وضعیت آپاچی رو با دستور زیر چک کنید.

systemctl status apache2

اگر که خروجی به صورت بالا نبود، از ‍‍‍systemctl برای اجرا شدن آپاچی استفاده کنید:

sudo systemctl start apache2

همچنین برای اجرا شدن آپاچی بعد از روشن شدن سیستم بهتره که آپاچی رو فعال کنید:

sudo systemctl enable apache2

با دستور زیر میتونید ورژن آپاچی رو چک کنید:

apache2 -v

خروجی به صورت زیر است:

Server version: Apache/2.4.38 (Ubuntu)
Server built:   2019-04-03T18:31:46

حالا برای اطمینان از اجرا شدن آپاچی توی مرورگر خودتون یکی از آدرس های 127.0.0.1 یا localhost رو وارد کنید. خروجی باید به صورت زیر باشه:

اگر درخواست رد شد یا انجام نشد، ممکنه که iptables فایروال از درخواست‌های ورودی پورت 80 جلوگیری میکند. برای باز کردن پورت TCP 80 باید دستور زیر رو اجرا کنید:

sudo iptables -I INPUT -p tcp --dport 80 -j ACCEPT

اگر شما از UFW فایروال استفاده میکنید دستور زیر را برای بازکردن پورت TCP 80 استفاده کنید:

sudo ufw allow http

خب حالا ما نیاز داریم که www-data رو هم به عنوان document root تنظیم کنیم.

sudo chown www-data:www-data /var/www/html/ -R

گام سوم: نصب MariaDB Database Server

MariaDB یک جایگزین برای MySQL هستش.

برای نصب روی اوبونتو 19.04 دستور زیر رو توی ترمینال وارد کنید:

sudo apt install mariadb-server mariadb-client

بعد از نصب MariaDB خودکار راه‌اندازی میشه. برای چک کردن وضعیت از دستور زیر استفاده میکنیم:

systemctl status mariadb

خروجی باید به شکل زیر باشه:

اگر که برای شما به درستی کار نکرد دستور زیر رو وارد کنید:

sudo systemctl start mariadb

برای فعال کردن اجرای خودکار هنگام روشن کردن دستگاه، از دستور زیر استفاده کنید:

sudo systemctl enable mariadb

اکنون اسکریپت امنیتی نصب پست رو اجرا کنید:

sudo mysql_secure_installation

در قسمت اول:

برای این قسمت فقط دکمه‌ی Enter را بزنید.

سپس سوالاتی که از شما پرسیده خواهد شد رو به ترتیب زیر پاسخ بدید:

Disallow root login remotely? [Y/n] Y or Press Enter

به طور پیشفرض بسته MaraiDB در اوبونتو از unix_socket برای تأیید اعتبار ورود به سیستم استفاده می کند که اساسا بدین معنی است که شما می توانید از نام کاربری و رمز عبور سیستم عامل برای ورود به کنسول MariaDB استفاده کنید.بنابراین شما می توانید دستور زیر را برای وارد شدن بدون ارائه رمز عبور root، میتوانیدMariaDB را اجرا کنید.

sudo mariadb -u root

برای خارج شدن دستور زیر رو وارد کنید:

exit;

ورژن MariaDB server رو با دستور زیر چک کنید:

mariadb --version

خروجی باید چیزی شبیه زیر باشد:

mariadb  Ver 15.1 Distrib 10.3.13-MariaDB, for debian-linux-gnu (x86_64) using readline 5.2

گام چهارم: نصب PHP7.2

خب حالا سراغ نصب PHP7.2 میریم. برای نصب باید دستور زیر رو وارد کنید:

sudo apt install php7.2 libapache2-mod-php7.2 php7.2-mysql php-common php7.2-cli php7.2-common php7.2-json php7.2-opcache php7.2-readline

ماژول آپاچی php7.2 را فعال کنید سپس سرور آپاچی را راه اندازی مجدد کنید.

sudo a2enmod php7.2

sudo systemctl restart apache2

ورژن PHP نصب شده رو چک کنید:

php --version

باید خروجی زیر برای شما چاپ شود:

PHP 7.2.19-0ubuntu0.19.04.1 (cli) (built: Jun  4 2019 14:44:42) ( NTS )
Copyright (c) 1997-2018 The PHP Group
Zend Engine v3.2.0, Copyright (c) 1998-2018 Zend Technologies
    with Zend OPcache v7.2.19-0ubuntu0.19.04.1, Copyright (c) 1999-2018, by Zend Technologies

برای تست اسکریپتهای PHP با سرور Apache، ما باید یک فایل info.php را در محل root ایجاد کنیم.

sudo nano /var/www/html/info.php

سپس در نانو کد زیر را وارد کنید و بعد به ترتیب با Enter , Y , Ctrl+x فایل مورد نظر رو ذخیره کنید.

<?php phpinfo(); ?>

بعد از وارد کردن این کد از طریق یکی از دو آدرس زیر در مرورگر ورژن PHP رو چک کنید:

127.0.0.1/info.php

localhost/info.php

نحوه اجرای PHP-FPM با Apache

اساسا دو راه برای اجرای کد PHP با سرور وب Apache وجود دارد:

  • ماژول Apache PHP

  • PHP-FPM

در مراحل فوق، ماژول PHP7.2 برای مدیریت کد PHP استفاده می شود که معمولا خوب است. اما در بعضی موارد، شما باید PHP را با PHP-FPM اجرا کنید. اما چطور؟

اول باید ماژول PHP7.2 رو غیر فعال کنید.

sudo a2dismod php7.2

نصب بسته‌ی PHP-FPM.

sudo apt install php7.2-fpm

فعال کردن ماژول های proxy_fcgi و setenvif.

sudo a2enmod proxy_fcgi setenvif

فعال کردن فایل پیکربندی /etc/apache2/conf-available/php7.2-fpm.conf.

sudo a2enconf php7.2-fpm

آپاچی را راه‌اندازی مجدد کنید تا تغییرات اعمال شود.

sudo systemctl restart apache2

حالا اگر صفحه info.php را در مرورگر خود بازنویسی کنید، می بینید که API سرور به FPM / FastCGI تغییر کرده است، به این معنی که وب سرور آپاچی درخواست PHP را به PHP-FPM منتقل می کند.

خب تبریک میگم، شما با موفقیت LAMP رو نصب کردید.

گام پنجم: نصب phpMyAdmin

بعد از نصب کردن LAMP نیازه که مخزن universe رو فعال کنید.

sudo add-apt-repository universe

اکنون زمان نصب phpMyAdmin است.

sudo apt update

sudo apt install phpmyadmin php-mbstring php-gettext

حالا، وب سرور را انتخاب کنید که باید به طور خودکار برای اجرای phpMyAdmin پیکربندی شود. وب سرور را با فلش های UP / DOWN انتخاب کنید و کلید SPACEBAR را فشار دهید. هنگامی که وب سرور را انتخاب کردید، نماد * (ستاره) در مقابل آن مشاهده خواهید کرد. کلید TAB را فشار دهید تا OK را انتخاب کنید و دوباره کلید ENTER را بزنید تا ادامه یابد.

برای ادامه ENTER را فشار دهید.

برای پیکربندی‌ پایگاه داده برای phpMyAdmin باید Yes را انتخاب کنید.

رمزعبور MySQL رو برای phpMyAdmin وارد کنید.

پسورد رو دوباره وارد کنید.

بعد از نصب phpMyAdmin افزونه‌ی mbstring برای PHP رو فعال کنید و سپس آپاچی رو راه‌اندازی مجدد کنید.

sudo phpenmod mbstring

sudo systemctl restart apache2

ساختن کاربر اختصاصی برای دسترسی به داشبورد phpMyAdmin

برای این‌کار ابتدا به شل MySQL وارد شوید:

mysql -u root -p

دستور زیر رو برای ایجاد کاربر جدید وارد کنید.

CREATE USER 'phpmyadminuser'@'localhost' IDENTIFIED BY 'password';

به جای phpmyadminuser نام کاربری خود را وارد کنید و به جای password هم نام کاربری مورد نظر خود را.

سپس دستور زیر را برای دادن اجازه‌ به کاربر وارد نمایید.

GRANT ALL PRIVILEGES ON *.* TO 'phpmyadminuser'@'localhost' WITH GRANT OPTION;

در اخر با دستور زیر خارج شوید:

exit

برای دسترسی به داشبورد phpMyAdmin به آدرس زیر بروید و از نام کاربری و رمزعبوری که در مرحله‌ی قبل تعیین کرده‌اید استفاده نمایید.

برای امنیت سرور شما بهتره info.php رو حذف کنید.

sudo rm /var/www/html/info.php

امیدوارم براتون مفید بوده باشه.
